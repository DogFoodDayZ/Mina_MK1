===========================
 MK1 CORE → FRONTEND CONTRACT
===========================

1. /process  (POST)
-------------------
Input:
{
  "input": "string"
}

Accepted Response Formats:
{
  "output": { "reply": "string" }
}

{
  "output": "string"
}

{
  "reply": "string"
}

{
  "choices": [
    { "message": { "content": "string" } }
  ]
}

Error Fallback:
{
  "output": { "reply": "(error contacting core)" }
}


2. /status  (GET)
-----------------
Response:
{
  "core_temp": "string",
  "memory_bus": "string",
  "neural_cache": "string",
  "io_channels": "string",
  "level": "NORMAL" | "WARN" | "ERROR" | "CRITICAL"
}


3. /db/status  (GET)
--------------------
Response:
{
  "db_link": "string",
  "db_sync": "string",
  "db_latency": "string",
  "active_connections": "string",
  "read_ops": "string",
  "write_ops": "string",
  "cache_state": "string",
  "last_commit": "string",
  "level": "NORMAL" | "WARN" | "ERROR" | "CRITICAL"
}


4. Severity Levels (Shared)
---------------------------
Allowed values:
NORMAL
WARN
ERROR
CRITICAL

Anything else defaults to WARN.


5. Stability Requirements
-------------------------
- Always return valid JSON.
- Never return HTML or plaintext.
- All fields must be strings.
- Missing fields are allowed; UI will show "UNKNOWN".
- Missing endpoint results in "WAITING FOR CORE…" or "WAITING FOR DB…".
- Polling interval: 2000ms.
- Field names must match exactly.


6. Safe Extensions
------------------
- You may add extra fields to /status or /db/status.
- UI ignores unknown fields safely.
